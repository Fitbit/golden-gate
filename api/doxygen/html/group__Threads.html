◆ GG_Mutex_LockAutoCreate()

GG_Result GG_Mutex_LockAutoCreate ( GG_Mutex **  mutex)
related

Mutex locking/creation

This is useful for creating/locking mutexes that are used as singletons for example. if *mutex is NULL, the mutex will be created in a thread-safe manner (it is ok to have several threads try to create the same mutex at the same time), and will then be locked. Otherwise, it will just be locked.

Parameters
mutexPointer to a variable in which the mutex will be returned.
Returns
GG_SUCCESS if the mutex could be created, or an error code.
